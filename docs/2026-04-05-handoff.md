## 1. 当前任务目标

为 OpenHarness 构建 **桌面端 Electron Shell（HTTP+WebSocket 协议 MVP）**，使 Python Agent 能力通过桌面应用提供给用户。

**完成标准：**
- 前端能通过 WebSocket 与后端通信，实现完整对话（发消息 → agent 执行 → 流式返回 → 工具调用 → 结果渲染）
- 侧边栏支持会话管理（新建/列表/恢复/删除/重命名/搜索）
- SQLite 持久化存储 + OpenHarness JSON 快照双层存储
- 后端以 `import` 方式调用 OpenHarness（不通过 CLI）

**当前阻塞：** 浏览器沙箱网络隔离导致前端无法连接后端（CORS 已解决，但服务器进程被系统频繁 SIGKILL，需确认 nohup 进程是否存活）

## 3. 关键上下文

### 用户明确要求
- 以 `import` 方式调用 OpenHarness（非 CLI）
- 桌面 Shell 主导 product-state 数据
- 会话列表**必须显示标题，不能显示 ID**
- 先规则提取标题（零 LLM 成本），后续接 LLM 智能生成
- 遇到问题先自主排查再求助

### 架构决策
| 决策点 | 选择 | 原因 |
|---|---|---|
| 集成方式 | import 库 | 非 CLI（CLI 慢、难以复用 Python 对象） |
| 事件类型 | 完全复用 OpenHarness 原生 | 避免重复造轮子 |
| REST 层 | 最小化 | CRUD 仅用于会话管理，slash 命令走 WS |
| 存储 | 双层 | SQLite（结构化查询）+ JSON 快照（备份兼容） |
| 标题生成 | 规则提取 MVP | 首条 user 消息前 30 字，预留 LLM 接口 |

### 依赖
- `apps/host-python/requirements.txt` — FastAPI、uvicorn、websockets 13.1（需 `.venv/bin/python3`）
- OpenHarness 库通过 `sys.path.insert(0, vendor/OpenHarness/src)` 导入
- 端口：后端 8789，前端静态 8091

### 环境变量
- `DEEPSEEK_BASE_URL`、`DEEPSEEK_API_KEY` → 映射到 `OPENHARNESS_*`
- `OPENHARNESS_CONFIG_DIR`、`OPENHARNESS_DATA_DIR` → 指向 `.tmp/` 目录

## 4. 关键发现

### 事件映射（已验证全链路）
```
OpenHarness 原生          →  WS 事件
──────────────────────────────────────────
ready                     →  session.ready
AssistantTextDelta        →  assistant.delta
AssistantTurnComplete    →  assistant.complete
ToolExecutionStarted      →  tool.started
ToolExecutionCompleted   →  tool.completed
transcript_item          →  transcript.item
modal_request            →  approval.request / question.request
line_complete            →  session.run_complete
shutdown                 →  session.closed
```

### 端到端验证过的完整流程
```
前端输入 "搜索gemma 4的资讯"
  → WS session.submit
  → OpenHarness agent 执行
  → assistant.delta（流式文本输出）
  → tool.started → tool.completed（web_search、web_fetch）
  → session.run_complete
  → 消息自动保存 SQLite（14 条消息验证通过）
```

### SQLite 存储验证
- 2 条会话，14 条消息（含 user/assistant/tool_result）
- 自动标题测试：`检查磁盘空间` → 正确提取

### 运行环境特性
- 系统 exec 后台进程容易被 SIGKILL（超时后）
- 用 `nohup ... > /tmp/xxx.log 2>&1 &` 可绕过
- 后端健康检查：`curl -s http://127.0.0.1:8789/health`
- **websockets 13.1 和系统 Python 3.12 有兼容问题，必须用 `.venv/bin/python3`**

## 7. 风险与注意事项

### ⚠️ 易踩的坑
1. **重复 `let` 声明** — 之前因复制粘贴导致 567-568 行重复声明，引发 `SyntaxError` 使整个 JS 不执行。新增 JS 代码时注意不要重复声明变量。
2. **engine 属性名** — `engine.model` 不存在，必须是 `engine._model`。
3. **路径层级** — `Path(__file__).parents[4]` 才能正确指向项目根目录。
4. **Python 版本** — 后端必须用 `.venv/bin/python3`，系统 Python 3.12 与 websockets 13.1 不兼容。
5. **进程被杀** — exec 后台进程超时后会被 SIGKILL，务必用 nohup 方式启动。每次新连接可能需要重啟。
6. **CORS** — 前端 8091 请求后端 8789 是跨域，后端已加 `CORSMiddleware`。新增 REST 接口时不要漏掉。

### ✅ 已验证通过的方向（不建议改）
- OpenHarness 事件映射方案（完全复用原生类型）
- SQLite + JSON 双层存储架构
- `import` 调用 OpenHarness（非 CLI）
- 标题规则提取（前 30 字，去命令前缀、标点截断）
